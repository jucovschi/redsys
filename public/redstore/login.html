Generally, ALL the functionality of the Website should be available through RPC services. The Website should give access to some (but not all) functionality. Hence, when implementing some functionality, implement it in functions that you can reuse from both, Website-logic as well as webservice interfaces.
A great exercise would be defining the RPC methods in some schema format like WSDL. That would make it much easier to automatically generate the right calls and APIs for other languages.

Login


In this page users will only see the Login box. The "Have Session ID" should only be visible to "services" through RPC. But for the story line I included it here as well.
The idea behind the loging in using a session id is that a 3rd party service (e.g. Drupal website) would first login with their login e.g. user "drupal admin", will create a new project, assign a repository to them etc and then let RedStore generate a session id. This session ID will be given to a Web Editor and it will be able to only create/change files in the project created earlier by the "drupal admin" user.
The Session IDs should expire after some time.
